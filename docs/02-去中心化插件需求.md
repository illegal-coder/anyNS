# 去中心化插件需求

## 1. 插件总体要求

所有去中心化解析插件必须依附 PowerDNS 主链路，通过统一插件接口被调用。插件不得直接接管主 DNS 入口，也不得依赖 CoreDNS 承载主解析逻辑。

每个插件必须具备：

- 查询解析能力。
- TTL 和缓存策略。
- 超时控制。
- RPC/API 故障隔离。
- 审计日志输出。
- 安全标签输出。
- 标准 DNS RR 映射。
- 与 ICANN 解析的冲突策略。

## 2. 插件优先级

### P1 样板插件

#### Handshake/HNS

优先级最高，作为首个插件实现。

原因：

- 最接近 DNS 根/权威解析语义。
- 适合验证替代根路由。
- 适合验证 PowerDNS hook 和缓存。
- 适合验证冲突策略和 DNSLog。

首版要求：

- 支持 HNS 域名查询。
- 支持 A/AAAA/NS/DS/TXT 等基础记录映射。
- 支持本地 hsd/hnsd 或远端 resolver 后端。
- 支持无记录、超时、节点不可用降级。

### P2 Wave 1 核心插件

#### ENS

要求：

- 支持 `.eth` 解析。
- 支持地址、contenthash、文本记录。
- 可映射到 `A/AAAA/TXT/URI/WALLET`。
- 支持 Ethereum RPC 或索引服务。

#### Namecoin `.bit`

要求：

- 支持 Namecoin `.bit`。
- 支持 Namecoin JSON 记录映射。
- 与 d.id/.bit 明确区分。

#### Stacks BNS

要求：

- BNS 默认指 Stacks Bitcoin Name System。
- 支持 zonefile/DID/命名空间相关记录。
- 支持映射到标准 RR 和扩展 API。

#### PNS-Polkadot

要求：

- 独立于 PulseChain PNS 实现。
- 支持 Polkadot 生态域名、地址和文本记录。
- 支持插件级启停和优先级配置。

#### PNS-PulseChain

要求：

- 独立于 Polkadot PNS 实现。
- 支持 PulseChain 生态域名、地址和文本记录。
- 支持与其他 EVM 类插件复用适配层。

#### Unstoppable Domains

要求：

- 支持常见 Web3 TLD。
- 支持钱包地址、文本记录、内容寻址。
- 钱包地址优先映射为 `WALLET`。

### P2 Wave 2 主流扩展

- Solana SNS：支持 `.sol` 和常见记录。
- SPACE ID：支持 `.bnb`、`.arb`、`.eth` 相关解析能力。
- TON DNS：支持 `.ton`、wallet、site、storage、next resolver。
- Tezos Domains：支持地址、文本和反向解析。
- Aptos Names：支持 Aptos 生态名称解析。
- SuiNS：支持 Sui 生态名称解析。

### P2 Wave 3 生态兼容

- Freename/FNS。
- RIF Name Service/RNS。
- FIO Handle。
- OpenAlias。
- ADA Handle。
- d.id/.bit。

这些插件作为生态兼容层实现，不应影响 P0-P2 Wave 1 的核心解析稳定性。

## 3. 命名冲突策略

- ICANN 合法域名默认优先正常递归。
- 替代根和去中心化命名必须显式命中策略。
- 同后缀多插件冲突时按租户、客户端视图和插件优先级决策。
- `.bit` 默认不自动合并 Namecoin 与 d.id。
- 所有冲突都必须写入审计日志。

## 4. 记录映射策略

标准 RR 优先：

- 地址记录映射到 `A/AAAA` 或 `WALLET`。
- 内容寻址映射到 `URI` 或 `TXT`。
- 服务入口映射到 `SRV/HTTPS/SVCB`。
- 邮件记录映射到 `MX`。
- 证书和安全记录映射到 `TLSA/CAA`。

扩展 API：

- 头像、社交资料、链上身份、复杂 profile 等不适合 DNS RR 的字段，通过 HTTP API 暴露。

## 5. 插件验收

每个插件必须验证：

- 成功解析。
- 无记录返回。
- 上游 RPC/API 超时。
- 上游返回错误。
- 缓存命中。
- 缓存过期。
- 审计日志。
- 安全标签。
- 同后缀冲突。
- ICANN 递归不受影响。
