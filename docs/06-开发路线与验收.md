# 开发路线与验收

> 2026-06-18 执行优先级已按新增需求调整。近期工作以
> [`10-当前高优先级路线.md`](10-当前高优先级路线.md) 为准：
> SOA/单标签 HNS 顶级权威区与证书/私有信任根管理为 P0，
> Cloudflare 风格控制面和 GitHub Actions 为 P1。原 P0-P4 架构顺序继续作为总体基线。

## 1. 阶段总览

开发顺序固定为：

1. P0 基础环境。
2. P1 单一插件。
3. P2 插件并联。
4. P3 安全防护。
5. P4 文档与交付。

该顺序不能反转。安全防护必须建立在基础解析和插件并联稳定之后，避免过早影响解析正确性。

## 2. P0 基础环境

交付物：

- PowerDNS Recursor。
- PowerDNS Authoritative。
- 插件运行时骨架。
- 管理 API 骨架。
- DNSLog 管线。
- 本机一键脚本。
- Docker Compose。
- 基础配置样例。

验收：

- ICANN 公网域名可解析。
- DNSSEC 可验证。
- 缓存有效。
- PowerDNS 权威区可用。
- `WALLET/TYPE262`、`HTTPS/SVCB` 样例可查询。
- 管理 API 健康检查可用。

## 3. P1 单一插件

交付物：

- HNS 插件。
- 插件模板。
- 插件配置样例。
- HNS 测试用例。

验收：

- HNS 插件从 PowerDNS 主链路触发。
- 成功解析返回标准 RR。
- 无记录返回正确 RCODE。
- 上游超时可降级。
- 缓存命中有效。
- DNSLog 记录完整。

## 4. P2 插件并联

交付物：

- Wave 1 插件。
- Wave 2 插件。
- Wave 3 插件。
- 统一路由策略。
- 同后缀冲突策略。
- 插件缓存隔离。
- RPC/API 故障隔离。

验收：

- 多插件可并联启用。
- 插件优先级稳定。
- 同后缀冲突可按策略处理。
- 单插件故障不拖垮主解析。
- ICANN 递归不受影响。
- 每个插件均输出审计日志。

## 5. P3 安全防护

交付物：

- 风险评分引擎。
- DNS tunneling 检测。
- DGA/高熵检测。
- NXDOMAIN flood 检测。
- DNS rebinding 检测。
- 异常 RR 检测。
- 外部 DNSLog/OOB 回调平台拒绝开关、规范化平台域名列表和管理 UI。
- 蜜罐转发。
- CoreDNS 可选安全 profile。

验收：

- 攻击样本可触发对应规则。
- 拦截动作符合策略。
- 外部 DNSLog 平台域名 exact/subdomain、大小写、尾点、IDNA、标签边界、
  关闭模式、allowlist 优先级和非法列表项均有自动化覆盖。
- DNSLog 字段完整。
- 蜜罐 API 可收到事件。
- 蜜罐失败时进入队列并重试。

## 6. P4 文档与交付

交付物：

- 项目需求书。
- 架构需求文档。
- 插件需求文档。
- 接口与数据模型文档。
- 安全与蜜罐文档。
- 部署与对接文档。
- 验收文档。

验收：

- 文档覆盖用户需求的全部 6 项。
- 文档明确 PowerDNS/CoreDNS 分工。
- 文档明确插件优先级。
- 文档明确一站式部署流程。
- 文档可指导后续开发，不需要重新决策核心架构。

## 7. 总体验收矩阵

| 要求 | 验收证据 |
| --- | --- |
| 主流去中心化 NS 插件 | 插件清单、优先级、测试用例 |
| PowerDNS + CoreDNS | 架构图、配置、部署 profile |
| AdGuard/审计 DNS 兼容 | 上游配置和联调记录 |
| DNSLog 蜜罐 API | API 测试、失败重试测试 |
| DNS 攻击拦截 | 攻击样本和规则命中记录 |
| 一站式部署 | 本机脚本和 Compose 空环境部署记录 |
| SOA 与 HNS 顶级权威区 | apex SOA/NS/glue、serial、IDNA、权威/递归 wire 与 Selenium 测试 |
| 证书与私有信任根 | ACME/private-ca 模式、密钥非披露、签发/续期/吊销/TLSA/备份测试 |
| GitHub Actions | Go/前端构建测试、shell/Compose 校验与无密钥构建产物 |
