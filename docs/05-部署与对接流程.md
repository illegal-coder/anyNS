# 部署与对接流程

## 1. 部署目标

anyNS 必须提供两种一站式部署方式：

- 本机一键部署。
- Docker Compose 部署。

两种部署方式都必须能完成：

- PowerDNS Recursor 启动。
- PowerDNS Authoritative 启动。
- 插件运行时启动。
- 管理 API 启动。
- DNSLog/蜜罐转发服务启动。
- 基础健康检查。
- 样例解析验证。

## 2. 本机一键部署流程

步骤：

1. 检查操作系统、权限和依赖。
2. 检查端口占用：53/udp、53/tcp、8080、8081 等。
3. 生成 PowerDNS Recursor 配置。
4. 生成 PowerDNS Authoritative 配置。
5. 生成插件运行时配置。
6. 生成管理 API 配置。
7. 生成 DNSLog/蜜罐转发配置。
8. 写入测试区和现代 RR 样例。
9. 启动服务。
10. 执行健康检查。
11. 执行 ICANN 公网解析测试。
12. 执行 HNS 样例插件测试。
13. 输出 AdGuard 上游配置。
14. 输出蜜罐 API 配置说明。

## 3. Docker Compose 部署流程

默认服务：

- `pdns-recursor`
- `pdns-authoritative`
- `anyns-plugin-runtime`
- `anyns-admin-api`
- `anyns-log-forwarder`

可选 profile：

- `coredns-security`
- `monitoring`

独立 PostgreSQL/gpgsql 模式：

```bash
export PDNS_POSTGRES_PASSWORD='replace-with-a-secret'
export PDNS_AUTH_API_KEY='replace-with-a-secret'
export PDNS_AUTH_WEBSERVER_PASSWORD='replace-with-a-secret'
docker compose -f docker-compose.gpgsql.yml up -d --wait
```

该拓扑默认将 DNS 和 API 分别绑定到回环地址的 `5301` 和 `8085`
端口，并使用 `anyns-pgsql-data` 持久化 PostgreSQL。升级前可使用
`pg_dump --clean --if-exists` 生成逻辑备份；恢复时先停止
`pdns-authoritative`，使用 `psql --set=ON_ERROR_STOP=1` 导入后再启动并验证
权威查询。自动化门禁为：

```bash
bash tests/acceptance/docker-gpgsql-backup-restore.sh
```

要求：

- 使用 `.env` 管理端口、密钥、上游 DNS、蜜罐 URL。
- 使用数据卷持久化 PowerDNS 数据和插件缓存。
- 提供 healthcheck。
- 提供日志目录。
- 支持镜像版本锁定。
- 支持升级回滚。

## 4. PowerDNS 配置要求

Recursor：

- 配置 ICANN 根提示或上游递归。
- 启用缓存。
- 启用 DNSSEC 验证。
- 启用 Lua hook 或等价扩展机制。
- 配置查询日志和审计日志。

Authoritative：

- 配置本地权威区。
- 配置测试区。
- 支持 Remote Backend 或 SQL Backend。
- 支持现代 RR 和未知 RR 类型兼容表达。

## 5. AdGuard 对接

对接方式：

1. 将 anyNS 配置为 AdGuard 上游 DNS。
2. 如启用 DoH/DoT，配置对应上游地址。
3. 保留 AdGuard 现有过滤策略。
4. 在 anyNS 侧开启 AdGuard 来源视图。
5. 验证 ICANN 域名解析。
6. 验证去中心化域名解析。
7. 验证拦截事件和日志。

验收：

- AdGuard 查询可以转发到 anyNS。
- anyNS 可以返回公网域名响应。
- anyNS 可以触发去中心化插件。
- anyNS 可以生成 DNSLog。
- AdGuard 过滤不被破坏。

## 6. 蜜罐对接

配置项：

- 蜜罐 API URL。
- API Key。
- HMAC Secret。
- 批量大小。
- 请求超时。
- 重试次数。
- 失败队列目录。

流程：

1. 配置蜜罐 API 地址和密钥。
2. 发送连通性测试事件。
3. 触发安全规则。
4. 检查 DNSLog。
5. 检查蜜罐事件。
6. 模拟 API 失败。
7. 验证失败队列和重试。

## 7. 升级与回滚

升级前：

- 备份配置。
- 备份 PowerDNS 数据。
- 备份插件配置。
- 记录镜像版本。

升级中：

- 先升级插件运行时。
- 再升级管理 API。
- 最后升级 PowerDNS 配置。

回滚：

- 回退镜像版本。
- 恢复配置。
- 恢复数据库或 zone 数据。
- 清理不兼容缓存。
- 执行验收测试。
