# 接口与数据模型

## 1. 插件接口

统一插件接口：

```text
resolve(qname, qtype, context) -> result
```

### 输入字段

```json
{
  "qname": "example.hns",
  "qtype": "A",
  "context": {
    "trace_id": "string",
    "client_ip": "192.0.2.10",
    "client_view": "default",
    "tenant": "default",
    "transport": "udp",
    "protocol": "dns",
    "policy_tags": ["string"]
  }
}
```

### 输出字段

```json
{
  "rrset": [
    {
      "name": "example.hns.",
      "type": "A",
      "ttl": 300,
      "value": "203.0.113.10"
    }
  ],
  "rcode": "NOERROR",
  "ttl": 300,
  "source_plugin": "hns",
  "confidence": "authoritative",
  "security_tags": ["decentralized", "hns"],
  "raw_record": {},
  "audit_metadata": {}
}
```

## 2. 路由策略模型

```yaml
routes:
  - name: hns-default
    match:
      suffixes: [".hns"]
      client_views: ["default"]
    plugin: hns
    priority: 100
    fallback: icann-recursive
  - name: ens-default
    match:
      suffixes: [".eth"]
    plugin: ens
    priority: 90
    fallback: nxdomain
```

要求：

- 支持后缀匹配。
- 支持租户规则。
- 支持客户端视图。
- 支持显式域名规则。
- 支持替代根优先级。
- 支持安全标签参与决策。
- 默认不得破坏 ICANN 递归。

## 3. 记录类型模型

必须支持：

- `A`
- `AAAA`
- `CNAME`
- `TXT`
- `MX`
- `NS`
- `SRV`
- `URI`
- `HTTPS`
- `SVCB`
- `TLSA`
- `CAA`
- `DS`
- `DNSKEY`
- `WALLET`

`WALLET` 要求：

- 按 IANA DNS RR type 262 处理。
- 支持原生 mnemonic。
- 支持 `TYPE262` RFC3597 兼容表达。
- 支持在 PowerDNS 未原生识别时以未知 RR 类型存储或转发。

## 5. 蜜罐 API

Endpoint：

```http
POST /api/v1/dns-events
```

Headers：

```http
Authorization: Bearer <api-key>
X-anyNS-Signature: <hmac-sha256>
X-anyNS-Idempotency-Key: <trace-id-or-event-id>
Content-Type: application/json
```

请求体：

```json
{
  "events": [
    {
      "timestamp": "2026-06-01T00:00:00Z",
      "trace_id": "string",
      "client_ip": "192.0.2.10",
      "qname": "tunnel.example",
      "qtype": "TXT",
      "risk_level": "high",
      "action": "forward_to_honeypot",
      "matched_rule": "dns-tunnel-high-entropy",
      "source_plugin": "security"
    }
  ]
}
```

响应：

```json
{
  "accepted": 1,
  "rejected": 0,
  "retry_after_seconds": 0
}
```

## 6. 管理 API

建议接口：

- `GET /healthz`
- `GET /readyz`
- `GET /metrics`
- `GET /api/v1/plugins`
- `POST /api/v1/plugins/{name}/enable`
- `POST /api/v1/plugins/{name}/disable`
- `POST /api/v1/policies/reload`
- `POST /api/v1/cache/flush`
- `GET /api/v1/audit/events`
- `GET /api/v1/honeypot/status`

管理 API 必须支持认证、审计和最小权限。

### PowerDNS DNSSEC 与记录管理

- `GET /api/v1/powerdns/authoritative/zones/{id}`
- `PATCH /api/v1/powerdns/authoritative/zones/{id}/rrsets`
- `GET /api/v1/powerdns/authoritative/zones/{id}/cryptokeys`
- `POST /api/v1/powerdns/authoritative/zones/{id}/cryptokeys`
- `DELETE /api/v1/powerdns/authoritative/zones/{id}/cryptokeys/{key_id}`
- `POST /api/v1/powerdns/authoritative/zones/{id}/derive-ds`

### ACME 证书与 TLSA

- `GET /api/v1/certificates/orders`
- `POST /api/v1/certificates/orders`
- `GET /api/v1/certificates/orders/{id}`
- `GET /api/v1/certificates/orders/{id}/certificate`
- `POST /api/v1/certificates/orders/{id}/renew`
- `POST /api/v1/certificates/orders/{id}/revoke`
- `POST /api/v1/certificates/tlsa`

证书作业状态为 `pending`、`validating`、`finalizing`、`issued`、`failed` 或 `revoked`。创建操作必须提供幂等键；API 不返回账户私钥或证书私钥。
