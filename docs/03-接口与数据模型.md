# 接口与数据模型

## 1. 插件接口

统一插件接口：

```text
resolve(qname, qtype, context) -> result
```

### 输入字段

```json
{
  "qname": "example.hns",
  "qtype": "A",
  "context": {
    "trace_id": "string",
    "client_ip": "192.0.2.10",
    "client_view": "default",
    "tenant": "default",
    "transport": "udp",
    "protocol": "dns",
    "policy_tags": ["string"]
  }
}
```

### 输出字段

```json
{
  "rrset": [
    {
      "name": "example.hns.",
      "type": "A",
      "ttl": 300,
      "value": "203.0.113.10"
    }
  ],
  "rcode": "NOERROR",
  "ttl": 300,
  "source_plugin": "hns",
  "confidence": "authoritative",
  "security_tags": ["decentralized", "hns"],
  "raw_record": {},
  "audit_metadata": {}
}
```

## 2. 路由策略模型

```yaml
routes:
  - name: hns-default
    match:
      suffixes: [".hns"]
      client_views: ["default"]
    plugin: hns
    priority: 100
    fallback: icann-recursive
  - name: ens-default
    match:
      suffixes: [".eth"]
    plugin: ens
    priority: 90
    fallback: nxdomain
```

要求：

- 支持后缀匹配。
- 支持租户规则。
- 支持客户端视图。
- 支持显式域名规则。
- 支持替代根优先级。
- 支持安全标签参与决策。
- 默认不得破坏 ICANN 递归。

## 3. 记录类型模型

必须支持：

- `A`
- `AAAA`
- `CNAME`
- `TXT`
- `MX`
- `NS`
- `SRV`
- `URI`
- `HTTPS`
- `SVCB`
- `TLSA`
- `CAA`
- `DS`
- `DNSKEY`
- `WALLET`

`WALLET` 要求：

- 按 IANA DNS RR type 262 处理。
- 支持原生 mnemonic。
- 支持 `TYPE262` RFC3597 兼容表达。
- 支持在 PowerDNS 未原生识别时以未知 RR 类型存储或转发。

## 5. 蜜罐 API

Endpoint：

```http
POST /api/v1/dns-events
```

Headers：

```http
Authorization: Bearer <api-key>
X-anyNS-Signature: <hmac-sha256>
X-anyNS-Idempotency-Key: <trace-id-or-event-id>
Content-Type: application/json
```

请求体：

```json
{
  "events": [
    {
      "timestamp": "2026-06-01T00:00:00Z",
      "trace_id": "string",
      "client_ip": "192.0.2.10",
      "qname": "tunnel.example",
      "qtype": "TXT",
      "risk_level": "high",
      "action": "forward_to_honeypot",
      "matched_rule": "dns-tunnel-high-entropy",
      "source_plugin": "security"
    }
  ]
}
```

响应：

```json
{
  "accepted": 1,
  "rejected": 0,
  "retry_after_seconds": 0
}
```

## 6. 管理 API

### 6.1 认证与 scope

启用管理认证后，客户端使用：

```http
Authorization: Bearer <management-key>
```

未提供或提供无效凭据返回 `401`；凭据有效但缺少 endpoint 所需 scope 返回
`403`。管理密钥可以直接声明 scope，也可以引用配置中的角色模板。

细粒度 scope：

| 功能 | 读取 scope | 写入 scope |
| --- | --- | --- |
| 管理概览、控制面边界和密钥元数据 | `management:read` | `management:write`（保留，当前无写入 endpoint） |
| PowerDNS 状态、zone 和缓存 | `powerdns:read` | `powerdns:write` |
| 插件 | `plugins:read` | `plugins:write` |
| Runtime 缓存 | `cache:read` | `cache:write` |
| 配置和安全策略内容 | `config:read` | `config:write` |
| 证书作业（ACME/private-ca）和 TLSA 发布 | `certificates:read` | `certificates:write` |
| 策略重载 | 无独立读取 scope | `policy:write` |
| 审计事件和聚合 | `audit:read` | 无写入 endpoint |
| 蜜罐状态和失败队列 | `honeypot:read` | `honeypot:write` |

兼容 scope `read` 授予所有 `*:read` 权限，`write` 授予所有 `*:write`
权限；`admin` 和 `*` 授予全部权限。细粒度写入 scope 不自动授予对应读取
scope。管理认证关闭时，匿名 principal 使用兼容 `read`、`write` scope。

`GET /api/v1/management/keys` 只返回密钥 ID、scope、角色、生命周期、
客户端限制和轮换状态等元数据，不返回 token 内容。

证书作业状态为 `pending`、`validating`、`finalizing`、`issued`、`failed`
或 `revoked`。创建和续期操作必须提供幂等键；证书下载接口只返回公开证书链，
不返回 ACME 账户私钥、private-ca 根私钥或叶证书私钥。

`GET /api/v1/certificates/private-ca/root` 只返回私有根 CA 的 subject、
serial、有效期、SHA-256 指纹、SKI/AKI、KeyUsage 和根私钥文件权限状态等
元数据；不返回根证书 PEM、根私钥或任何叶证书私钥。

`GET /api/v1/certificates/private-ca/root/certificate` 是显式的根证书下载端点，
返回当前活动私有根 CA 的公开证书 PEM，供受控客户端安装信任。该端点要求
`certificates:read`，支持 `HEAD`，响应只包含单个 CA 证书 PEM，不返回根私钥、
叶证书私钥、证书存储路径或备份路径。下载根证书不表示公开 WebPKI 信任；
客户端仍必须由操作员显式安装并信任该根。

`PATCH /api/v1/certificates/private-ca/root` 目前只接受 `{"disabled": true|false}`。
禁用后，private-ca issuer 会拒绝新的叶证书签发；已签发证书的清单、下载和
作业状态查询仍可读取。

`POST /api/v1/certificates/private-ca/root/backup-status` 接受当前根 CA 的
`{"sha256_fingerprint":"..."}`，用于记录操作员已备份该根的证据。根元数据的
`backup_status.status` 为 `missing`、`current` 或 `stale`；该标记只持久化根证书
SHA-256 指纹和记录时间，不扫描备份系统、不返回路径、PEM 或私钥。

`POST /api/v1/certificates/private-ca/root/import` 接受 `certificate_pem` 和
`private_key_pem`，只导入自签名 CA、具备 certSign、SKI/AKI 匹配、未过期且私钥
与证书公钥匹配的根。导入后根私钥以 `0600` 写入，证书管理器会重新加载 issuer；
响应和审计只记录新根元数据和 SHA-256 指纹，不返回 PEM 或私钥。

`POST /api/v1/certificates/private-ca/root/rotate` 生成新的本地私有根 CA，
覆盖当前活动根并重新加载 issuer；之后的新 private-ca 叶证书由新根签发。
已有证书作业和证书链仍可读取，旧根备份标记会在元数据中显示为 `stale`。
响应和审计只记录新根元数据和 SHA-256 指纹，不返回 PEM、私钥或存储路径。

`GET /api/v1/certificates/private-ca/crl` 返回当前活动私有根签名的 PEM CRL。
CRL 只包含由当前根签发且已在 anyNS 作业中标记为 `revoked` 的叶证书 serial；
响应不包含证书链 PEM、根私钥或叶私钥。
如果 `certificates.crl_distribution_url` 配置为带路径的 HTTP(S) URL，且路径不是
`/api`、`/healthz`、`/readyz` 或 `/metrics` 等保留路径，Admin 服务会在同一路径
额外提供无需管理凭据的 PEM CRL，用于让新签发叶证书中的 CRL Distribution Point
直接可取。该公开路径只返回 CRL，不返回根证书、证书链、私钥或存储路径。

`derive-ds` 只根据调用方提供的 DNSKEY 计算 DS，不修改 PowerDNS，因此使用
`powerdns:read`。私钥只写入服务器端 `storage_dir`，任何管理 API 都不得返回。

### 6.3 Capability 合约

`GET /api/v1/capabilities` 返回版本、生成时间和每个前端功能的可用性：

```json
{
  "version": 1,
  "generated_at": "2026-06-12T00:00:00Z",
  "features": {
    "powerdns": {
      "available": true,
      "read": true,
      "write": false,
      "mode": "readonly",
      "reason": "write_not_available",
      "endpoints": [
        "GET /api/v1/powerdns/status",
        "GET /api/v1/powerdns/zones"
      ]
    }
  }
}
```

模式定义：

| Mode | 含义 | Reason |
| --- | --- | --- |
| `hidden` | principal 缺少功能读取 scope | `access_denied` |
| `unavailable` | 有读取权限，但后端或配置文件未配置 | `backend_not_configured` |
| `readonly` | 可读取，但缺少写入 scope 或配置不可写 | `write_not_available` |
| `readwrite` | 后端可用、配置可写且同时具有读写 scope | 省略 |

Capability 计算直接使用各 endpoint 的细粒度读取 scope，不额外要求
`management:read`。响应不得包含 API key、Bearer token 或其他密钥材料。

### 6.4 Dashboard 非披露规则

`GET /api/v1/dashboard` 本身要求 `management:read`，但该 scope 只允许读取
基础 Admin/Runtime 服务状态。其他字段按 scope 独立添加：

| Dashboard 字段 | 附加 scope |
| --- | --- |
| `services.powerdns` | `powerdns:read` |
| `plugins` | `plugins:read` |
| `cache` | `cache:read` |
| `audit_summary`、`recent_events` | `audit:read` |
| `configuration` | `config:read` |

缺少附加 scope 时字段必须从 JSON 响应中省略，而不是返回空对象或脱敏后的
功能数据。Admin 启用 Runtime 代理时，插件和缓存控制面请求转发到 Runtime，
并继续使用调用方的 Authorization header。
