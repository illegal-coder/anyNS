# 当前高优先级路线

日期：2026-06-19

本文件覆盖原 P0-P4 路线中的近期执行顺序。原架构原则不变：PowerDNS 是主解析与管理路径，CoreDNS 仍是可选安全 profile。

## P0：SOA、根域名与顶级域名解析

目标是把 HNS 名称按真正的单标签顶级权威区管理，而不是把 `.hns` 当作公开 DNS 子区。

交付范围：

- 单标签 HNS/IDNA Zone 创建、读取、删除和冲突检查。
- Zone apex 的 SOA、NS、A/AAAA glue 与 serial 生命周期。
- SOA 字段编辑、默认值、边界校验、原子更新和审计。
- HNS 链上 NS/GLUE 展示与 PowerDNS 权威记录的一致性检查。
- Authoritative、Recursor、hnsd、DoH、DoT 的顶级域名查询链路。
- Cloudflare 风格的 Zone 概览、DNS 记录、SOA/DNSSEC 管理体验。

验收：

- `example.` 和 Unicode 单标签 Zone 可返回正确 SOA/NS/A/AAAA。
- `.hns` 只作为用户可读别名出现，不进入 PowerDNS Zone 主键。
- `www.example` 不能被误创建为 HNS 顶级 Zone。
- SOA serial 更新、错误输入、越区记录和权限矩阵都有自动化测试。
- Selenium 覆盖桌面与移动端创建 Zone、编辑记录、查看 SOA/DNSSEC。

## P0：证书与信任根管理

现有 ACME DNS-01 保留。新增能力采用 clean-room 实现，不复制 `eskimo/certy` 源码或其 PKI.js 副本。

信任模式必须明确分离：

1. `acme`：面向公开 WebPKI 可验证名称。
2. `private-ca`：由部署者管理私有根 CA，适用于 HNS、内网和受控客户端。
3. `browser-local`：可选的浏览器本地自签名叶证书与 TLSA 生成；私钥不离开浏览器。该模式不是根 CA。

交付范围：

- 私有根 CA 创建、导入、禁用、轮换、备份状态和证书指纹。
- 使用活动根 CA 签发叶证书，支持 SAN、受控 wildcard、有效期和用途约束。
- 证书清单、到期状态、下载、续期、吊销、TLSA 生成/发布和审计。
- 根私钥与叶私钥 `0600` 存储、API 不返回私钥、日志不记录密钥或 challenge。
- 明确显示“公开可信”“私有信任”“自签名”的差异。

验收：

- 根 CA 具备正确的 BasicConstraints、KeyUsage、SKI/AKI 和随机 serial。
- 叶证书不能继续签发 CA，SAN/有效期不得越过根证书策略。
- 私钥非披露、权限、备份恢复、轮换、吊销和并发签发均有测试。
- DANE/TLSA 仅在 DNSSEC 信任链或明确私有信任策略下使用。

当前证据：

- [x] `private-ca` issuer mode 已与 ACME 显式分离；clean-room 私有根签发器使用 Go `crypto/x509` 创建/加载本地根 CA，根私钥以 `0600` 存储，叶证书包含 SAN/serverAuth、SKI/AKI、随机 serial 和根链。
- [x] `bash tests/acceptance/private-ca-certificates.sh` 使用隔离 Compose profile 构建 Admin 镜像并验证 `private-ca` 模式签发、证书下载不含私钥、SAN/serverAuth/非 CA 约束、根链验证、根/叶私钥 `0600`、重启持久化和容器内备份恢复。
- [x] `GET /api/v1/certificates/private-ca/root` 返回私有根 CA subject、serial、有效期、SHA-256 指纹、SKI/AKI、KeyUsage 和根私钥权限状态，不返回 PEM 或私钥材料。
- [x] `PATCH /api/v1/certificates/private-ca/root` 支持禁用/启用私有根 CA；禁用状态持久化并阻止新的 private-ca 叶证书签发，管理审计不记录 PEM 或私钥。

- Zone 首页与状态。
- DNS Records、SOA、NS/Glue、DNSSEC。
- SSL/TLS、ACME、Private CA、证书清单。
- 插件、缓存、安全、审计和配置。
